# Session 040 — 2026-06-14 — авто-уведомление магазина о новом заказе (email, любой канал)
← [session 039](./session_039_2026-06-14_manager-role.md)

## Session Intent
Проблема: заказ сохраняется в БД, но уведомление владельца зависело от того, нажмёт ли клиент дип-линк
(tg/wa/mailto) на странице успеха. Закрыл вкладку — заказ виден только в админке → владельцу пришлось бы
мониторить её 24/7. Нужно надёжное авто-уведомление.

## Решение (выбор пользователя)
Email, НЕ Telegram-бот. Причина пользователя: Telegram у него работает только через VPN; ставить VPN/прокси
на сервер ради api.telegram.org — муторно. Почта проще, без вопроса достижимости, SMTP уже заложен, и при
почтовом клиенте на телефоне письмо приходит как пуш. (Уточнил: бот шлёт С СЕРВЕРА, не с телефона — но
выбор email остаётся разумным и простым.)

## What works now (evidence)
- `manage.py check` → 0 issues; `manage.py test` → **44/44 OK** (новый тест проходит).
- Получатель уведомления = `SiteSettings.email` (поле «Email для заказов», редактируется в админке).
- В dev письмо уходит в консоль (EMAIL_BACKEND=console); на проде — SMTP из env (.env.example: EMAIL_*).

## Заметки / операционное
- ⚠️ Если `SiteSettings.email` пуст — уведомление НЕ уйдёт (логируется warning). Заполнить в админке.
- Дип-линки на странице успеха остались как НЕОБЯЗАТЕЛЬНЫЙ бонус для клиента; владелец узнаёт о заказе
  независимо от них. Текст success-страницы можно опц. смягчить («заказ принят, свяжемся») — НЕ делал.
- Без новых зависимостей. Telegram-уведомление можно добавить позже (если сервер дотянется до Telegram).
