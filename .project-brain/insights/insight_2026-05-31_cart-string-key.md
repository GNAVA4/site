# Insight — корзина не должна хранить состояние в склеенной строке-ключе
_2026-05-31, session 001_

## Что было
`session['cart']` хранил `{ "f{product_id}_{size}": qty }`, а `cart_detail` разбирал
ключ через `key.split('_')`. Два бага:
1. Размер с `_` (например «200_300») → split даёт >2 частей → ValueError → весь ключ
   уходит в p_id → `Product.objects.get` падает → **500 на всю корзину**.
2. `Product.objects.get(pk=...)` без `DoesNotExist` → удалили товар в админке →
   корзина клиента целиком рушится.

## Урок
Идентификатор позиции и её данные — разные вещи. Ключ можно использовать только как
непрозрачный id, а product_id/size/qty хранить явными полями и НИКОГДА не парсить ключ
обратно. Чтение товаров — батчем (`filter(pk__in=...)`), отсутствующие — молча пропускать.

## Где исправлено
`store/cart.py` (класс Cart) + `store/views.py` (cart_detail/add/remove/clear).
Проверено смоук-тестом: размер «200_300» сохраняется как size, ключ «id:200_300».
